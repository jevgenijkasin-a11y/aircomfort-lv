# Деплой на Garmtech (Plesk) после миграции с Supabase

Сайт теперь полностью самодостаточный: база — SQLite-файл, картинки — на диске.
Внешних сервисов нет. Всё живое (база + картинки) лежит в папке `data/`,
которая **не** в git — деплой кода её не трогает.

```
httpdocs/
├── data/
│   ├── aircomfort.db          <- база (товары, настройки, заявки, отзывы, визитки)
│   └── uploads/
│       ├── products/          <- фото товаров и слайдов
│       ├── product-images/
│       └── employee-photos/   <- фото сотрудников
└── ... (код из git)
```

## Порядок первого деплоя (важно: сначала данные, потом код)

1. **FTP: залить данные.** В Plesk → Files (или любым FTP-клиентом, доступы в
   Connection Info) загрузить локальную папку `data/` целиком в корень сайта
   (`httpdocs/data/`): файл `aircomfort.db` и папку `uploads/` (~60 МБ).

2. **Git: задеплоить код.** В Plesk → Git → Pull/Deploy (после push в GitHub).

3. **Пересобрать и перезапустить.** В Plesk → Node.js:
   - `NPM install` (зависимость Supabase удалена, ничего нового не нужно);
   - выполнить script `build`;
   - Restart App.

4. **Проверить:**
   - https://aircomfort.lv/lv — главная, слайды и товары с картинками;
   - каталог, страница товара, форма контактов;
   - /admin — вход по **новому** паролю (см. локальный файл
     `migration/admin-password.txt`; старый `admin1234` больше не работает);
   - после входа сменить пароль в админке на свой.

5. **Отключить старое:**
   - Vercel: удалить/запаузить проект (сайт живёт только на Garmtech);
   - Supabase: убедиться, что сайт работает несколько дней → удалить проект
     (полный экспорт данных сохранён локально в `migration/`).

## Как это устроено

- База: встроенный в Node.js модуль `node:sqlite` (нужен Node ≥ 22; на сервере
  24.18 — ок). Нативных модулей нет, компилировать нечего.
- Пароль админки хранится только как scrypt-хэш в таблице `auth` внутри БД.
  Секрет сессии генерируется автоматически там же.
- Все запросы к базе — только на сервере (роуты `/api/...` с проверкой сессии).
  Браузер к базе доступа не имеет.
- Загрузка картинок из админки пишет в `data/uploads/...`, раздача — через
  роут `/uploads/*` с кэшированием на год.
- Бэкап = стандартный Plesk Backup (файлы включают `data/`), либо просто
  скачать `data/` по FTP.

## Переменные окружения (все опциональны)

- `DATA_DIR` — путь к папке данных (по умолчанию `./data`);
- `ADMIN_SECRET` — свой секрет сессий вместо хранимого в БД;
- `ADMIN_PASSWORD` — используется один раз, только если в БД ещё нет хэша пароля;
- SMTP-переменные для почты — как раньше (`SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`,
  `SMTP_PORT`, `SMTP_TO`).
